# mihomo-config-builder v0.1.0

Initial public release of `mcb` with deterministic Mihomo config generation and NixOS automation support.

## Highlights

- CLI commands:
  - `mcb build -c profile.yaml -o config.yaml`
  - `mcb validate -f config.yaml`
  - `mcb diff -c profile.yaml --against old.yaml` (simplified)
- Multi-source input:
  - subscription URL(s)
  - local subscription file(s)
  - optional template file/URL
- Override engine:
  - YAML merge patches
  - JSON patch
  - strategy patch for rule prepend/append
- Deterministic output:
  - stable key ordering for clean diffs
  - reproducible YAML rendering
- Compatibility checks:
  - required top-level schema checks
  - fake-ip risk warnings
- v1 additions included:
  - JS hook runtime with fixed API (`mcbTransform(config, ctx)`)
  - built-in rule templates: `cn-direct`, `steam-direct-enhanced`

## Nix and Operations

- `flake.nix` package output for `mcb`
- NixOS module with systemd oneshot + timer
- Home Manager module example
- CI workflow for lint (`go vet`) and tests (`go test`)

## Known Notes

- `mcb diff` intentionally reports changed/no-change and leaves full patching to external diff tools.
- `nix flake check` may show non-blocking warnings for app metadata and custom flake output naming.

## Quick verification commands

```bash
nix shell nixpkgs#go -c go test ./...
nix shell nixpkgs#go -c go vet ./...
nix build .#mihomo-config-builder --no-link

nix shell nixpkgs#go -c go build -o bin/mcb ./cmd/mcb
./bin/mcb build -c examples/basic/profile.yaml -o /tmp/mcb-basic.yaml
./bin/mcb validate -f /tmp/mcb-basic.yaml
```
