# mihomo-config-builder v0.2.2

This patch release makes `subscriptions[].url` handle the same subscription payload formats already supported in mixed `nodes.txt` input.

## Highlights

- `subscriptions[].url` now normalizes remote payloads before merge:
  - Mihomo/Clash YAML subscriptions
  - plain node URI line subscriptions
  - base64-encoded node URI line subscriptions
- Remote node-link subscriptions now generate a synthetic Mihomo subscription shape automatically, so mixed profiles do not fail with `yaml root must be mapping`.
- Parsing behavior stays aligned between direct URL subscriptions and subscription URLs embedded inside `nodes.txt`.

- `go test ./...` passes
- `nix build .#mihomo-config-builder --no-link` should be run before cutting the tag
- `go vet ./...` should be run before cutting the tag
